1. Introducción y Contexto del Ejercicio

🎯 Propósito

Este ejercicio práctico está diseñado para desarrollar competencias fundamentales en la construcción de contenido digital, específicamente en el ámbito del periodismo web y la comunicación efectiva. Como estudiantes de Diseño Crossmedia, es esencial que dominen no solo los aspectos visuales y técnicos del diseño, sino también la creación de contenido de calidad que conecte con las audiencias.

🔑 Concepto Clave: El UX Writing no se trata solo de escribir bien, sino de escribir con propósito. Cada palabra debe servir a la experiencia del usuario y facilitar la comprensión del mensaje.

📋 Descripción del Ejercicio

Durante este ejercicio de 2 horas, trabajarás en la creación de 30 noticias distribuidas en 10 categorías temáticas. Cada categoría requiere un tono de comunicación específico, una estructura particular y un enfoque editorial diferenciado.

Las noticias se publicarán en un sitio web con diseño neumórfico, que ya está desarrollado y listo para que añadas tu contenido. Tu tarea es redactar contenido claro, preciso y adaptado a cada categoría, siguiendo los principios de UX Writing y escritura para personas.

⏱️ Duración y Modalidad

  • Tiempo total: 2 horas
  • Modalidad: Trabajo colaborativo
  • Plataforma: GitHub (control de versiones)
  • Entregable: 30 archivos HTML con noticias completas
⚠️ Importante: Este ejercicio simula un entorno de trabajo real en medios digitales. La gestión del tiempo, la calidad del contenido y la colaboración son factores críticos para el éxito.

2. Objetivos de Aprendizaje

🎯 Objetivo General

Desarrollar habilidades de construcción de contenido, UX Writing y escritura para personas, mediante la redacción de noticias adaptadas a diferentes audiencias y propósitos comunicativos.

🎯 Objetivos Específicos

1. Construcción de Contenido

  • Aplicar la estructura de pirámide invertida en la redacción periodística
  • Jerarquizar información según relevancia y necesidades del usuario
  • Integrar elementos multimedia (imágenes, citas, datos) de forma efectiva
  • Crear titulares y subtítulos que capten la atención sin sacrificar claridad

2. UX Writing

  • Escribir con claridad y concisión, eliminando ambigüedades
  • Adaptar el lenguaje según el contexto y la audiencia objetivo
  • Usar voz activa y lenguaje directo para mejorar la comprensibilidad
  • Optimizar la escaneabilidad del contenido mediante subtítulos y párrafos cortos

3. Escritura para Personas

  • Escribir pensando en las necesidades reales de los lectores
  • Evitar jerga innecesaria y términos confusos
  • Generar empatía a través del tono y estilo de comunicación
  • Priorizar la accesibilidad y la inclusión en el lenguaje

4. Colaboración y Trabajo en Equipo

  • Utilizar herramientas de control de versiones (Git/GitHub)
  • Trabajar en paralelo con otros estudiantes sin conflictos
  • Revisar y proporcionar retroalimentación constructiva
  • Mantener una comunicación efectiva con el equipo
✅ Competencias Desarrolladas: Al completar este ejercicio, habrás practicado habilidades esenciales para tu futuro profesional como diseñador crossmedia, incluyendo redacción, edición, adaptación de tono y trabajo colaborativo.

3. Instrucciones Técnicas Paso a Paso

Paso 1: Configuración Inicial

1.1. Clonar el Repositorio

git clone [URL-DEL-REPOSITORIO]
cd ejercicio_noticias_ucc

1.2. Crear tu Rama de Trabajo

Cada estudiante debe trabajar en su propia rama para evitar conflictos:

git checkout -b tu-nombre-apellido
Ejemplo: git checkout -b maria-rodriguez
⚠️ Nunca trabajes directamente en la rama main. Siempre crea una rama personal.

Paso 2: Crear las Carpetas de Trabajo

El repositorio ya incluye la estructura básica, pero necesitarás crear las subcarpetas para organizar tus noticias:

mkdir -p noticias/salud noticias/politica noticias/deportes
mkdir -p noticias/tecnologia noticias/cultura noticias/economia
mkdir -p noticias/educacion noticias/medio-ambiente
mkdir -p noticias/entretenimiento noticias/ciencia

Paso 3: Crear una Noticia

3.1. Duplicar la Plantilla

Para cada noticia, duplica el archivo template-noticia.html:

cp template-noticia.html noticias/salud/nueva-vacuna-covid.html

3.2. Editar el Contenido

Abre el archivo con tu editor de código y sigue estas indicaciones:

  1. Lee los comentarios HTML que dicen <!-- INSTRUCCIÓN: ... -->
  2. Reemplaza el lorem ipsum con tu contenido real
  3. Actualiza los metadatos:
    • Título de la página (<title>)
    • Meta description
    • Fecha de publicación
    • Autor (tu nombre)
    • Categoría
  4. Agrega imágenes relevantes:
    • Imagen destacada
    • Al menos 2 imágenes de apoyo
    • Escribe descripciones alt significativas
    • Incluye pies de foto con créditos
  5. Escribe el contenido siguiendo la estructura:
    • Primer párrafo: responde las 5W (Qué, Quién, Cuándo, Dónde, Por qué)
    • Cuerpo: desarrolla la información en orden de importancia
    • Incluye citas textuales si son relevantes
    • Usa subtítulos para organizar secciones
  6. Añade etiquetas al final del artículo (5-8 palabras clave)

3.3. Nomenclatura de Archivos

Usa nombres descriptivos en minúsculas, con guiones en lugar de espacios:

Correcto ✅ Incorrecto ❌
nueva-vacuna-covid-colombia.html noticia1.html
elecciones-presidenciales-2026.html Noticia Política.html
inteligencia-artificial-educacion.html mi_articulo.html

Paso 4: Guardar tus Cambios

4.1. Agregar Archivos al Área de Stage

git add noticias/salud/nueva-vacuna-covid.html
O para agregar todos los cambios:
git add .

4.2. Hacer Commit

Escribe un mensaje descriptivo que explique qué contiene el commit:

git commit -m "Añadida noticia sobre nueva vacuna COVID en Colombia"
Buenas Prácticas de Commits:
  • Haz commits frecuentes (idealmente uno por noticia)
  • Usa mensajes claros y descriptivos
  • Escribe en presente: "Añade" en lugar de "Añadido"
  • Sé específico sobre los cambios realizados

Paso 5: Subir tus Cambios

git push origin tu-nombre-apellido

Paso 6: Crear un Pull Request

  1. Ve al repositorio en GitHub
  2. Haz clic en "Compare & pull request"
  3. Escribe un título descriptivo
  4. Añade una descripción con la lista de noticias creadas
  5. Asigna el pull request al profesor

4. Guía de Buenas Prácticas de UX Writing

📝 Principios Fundamentales

1. Claridad sobre Todo

La claridad es el principio más importante del UX Writing. El lector debe entender tu mensaje sin esfuerzo.

Evita esto ❌ Mejor así ✅
"La implementación del nuevo paradigma educativo ha generado controversia en el sector académico" "El nuevo modelo educativo ha generado debate entre profesores y estudiantes"
"Se procederá a la realización de una evaluación exhaustiva" "Evaluaremos el proyecto en detalle"

2. Concisión

Elimina palabras innecesarias. Cada palabra debe aportar valor.

Evita esto ❌ Mejor así ✅
"En el momento actual" "Ahora" o "Actualmente"
"Con el objetivo de" "Para"
"A pesar del hecho de que" "Aunque"

3. Voz Activa

La voz activa es más directa y fácil de entender que la voz pasiva.

Voz Pasiva ❌ Voz Activa ✅
"La vacuna fue aprobada por la OMS" "La OMS aprobó la vacuna"
"El proyecto será implementado por el gobierno" "El gobierno implementará el proyecto"

4. Evita Jerga Innecesaria

Si debes usar términos técnicos, explícalos brevemente.

Evita: "La implementación del framework permitió optimizar el throughput"
Mejor: "El nuevo sistema permitió procesar más transacciones por segundo"

5. Usa Subtítulos Descriptivos

Los subtítulos ayudan a escanear el contenido y encontrar información rápidamente.

Ejemplo de estructura con subtítulos:
  • H1: "Nueva vacuna reduce hospitalizaciones en un 85%"
  • H2: "Resultados de los ensayos clínicos"
  • H2: "Disponibilidad en Colombia"
  • H2: "Qué esperar en los próximos meses"

6. Pirámide Invertida

Presenta la información más importante primero, seguida de detalles secundarios.

PRIMER PÁRRAFO: Las 5W
Qué, Quién, Cuándo, Dónde, Por qué
CUERPO: Detalles Importantes
Contexto, antecedentes, datos relevantes
FINAL: Información Complementaria
Detalles adicionales, perspectivas futuras

📏 Estructura de una Noticia

Título (H1)

  • Longitud: 50-70 caracteres
  • Características: Claro, conciso, informativo
  • Debe incluir: El hecho principal
  • Evitar: Clickbait, ambigüedad, preguntas retóricas
✅ Buen título: "Colombia lanza programa de energía solar para zonas rurales"
❌ Mal título: "No vas a creer lo que pasó con la energía en Colombia"

Subtítulo / Bajada

  • Longitud: 100-150 caracteres
  • Función: Complementar el título con información adicional
  • Debe incluir: Detalles que no cupieron en el título
Ejemplo de subtítulo: "El proyecto beneficiará a 50.000 familias en 200 municipios y reducirá costos de electricidad en un 40%"

Primer Párrafo (Lead)

  • Longitud: 60-80 palabras
  • Estructura: Responde las 5W
    • Qué: ¿Qué sucedió?
    • Quién: ¿A quién involucra?
    • Cuándo: ¿Cuándo ocurrió?
    • Dónde: ¿Dónde sucedió?
    • Por qué: ¿Por qué es importante?

Cuerpo del Artículo

  • Párrafos: Mínimo 5, idealmente 6-8
  • Longitud de párrafos: 40-60 palabras cada uno
  • Estructura: Una idea principal por párrafo
  • Elementos a incluir:
    • Contexto y antecedentes
    • Desarrollo del acontecimiento
    • Citas de fuentes relevantes
    • Datos y estadísticas
    • Implicaciones y perspectivas futuras

📸 Imágenes y Elementos Multimedia

Imagen Destacada

  • Debe representar visualmente el tema de la noticia
  • Alta calidad y resolución adecuada
  • Descripción alt detallada para accesibilidad
  • Pie de foto con contexto y créditos

Imágenes de Apoyo

  • Mínimo 2 imágenes adicionales
  • Complementan y amplían la información textual
  • Ubicadas estratégicamente para dividir bloques de texto
  • Cada imagen con su propio pie de foto

Citas Textuales

Las citas añaden credibilidad y voz humana a la noticia:

  • Usa comillas para citas directas
  • Identifica claramente al autor de la cita
  • Incluye el cargo o contexto relevante
  • Asegúrate de que la cita aporte valor
Ejemplo de cita bien estructurada:
"Este programa representa un paso fundamental hacia la transición energética del país", afirmó María García, ministra de Energía.

🏷️ Etiquetas (Tags)

Las etiquetas ayudan a categorizar y encontrar contenido relacionado:

  • Incluye 5-8 palabras clave relevantes
  • Usa palabras específicas, no genéricas
  • Piensa en cómo los usuarios buscarían este contenido
  • Incluye sinónimos y variaciones relevantes
Ejemplo para noticia sobre energía solar:
energía-solar, energías-renovables, zonas-rurales, electrificación, sostenibilidad, medio-ambiente, gobierno-colombia

5. Lineamientos de Comunicación por Categoría

Cada categoría de noticias tiene características específicas en cuanto a tono, estilo, estructura y enfoque editorial. A continuación, encontrarás lineamientos detallados para cada una de las 10 categorías.

❤️ 5.1. SALUD

Tono de Voz

  • Serio y profesional sin ser alarmista
  • Empático y sensible al tratar temas delicados
  • Informativo y educativo para empoderar al lector
  • Basado en evidencia científica y fuentes confiables

Tipo de Lenguaje

  • Términos médicos explicados en lenguaje sencillo
  • Evitar tecnicismos innecesarios
  • Usar analogías para conceptos complejos
  • Preferir "personas con diabetes" sobre "diabéticos" (lenguaje centrado en la persona)

Estructura Recomendada

  1. Titular: Qué avance/problema de salud y su impacto
  2. Lead: Información clave: qué, quién (afectados/investigadores), cuándo, dónde
  3. Contexto médico: Antecedentes de la enfermedad/tratamiento
  4. Evidencia científica: Estudios, datos, estadísticas
  5. Voces expertas: Citas de médicos, investigadores, autoridades sanitarias
  6. Impacto práctico: Qué significa para los pacientes/población
  7. Próximos pasos: Disponibilidad, recomendaciones, seguimiento

Palabras Clave Frecuentes

salud, tratamiento, prevención, pacientes, estudio, investigación, síntomas, diagnóstico, vacuna, terapia, bienestar, medicina, sistema-sanitario

Enfoque Editorial

  • Prioridad 1: Exactitud y verificación de datos médicos
  • Prioridad 2: Claridad para el público general
  • Prioridad 3: Contexto que ayude a tomar decisiones informadas
  • Evitar: Sensacionalismo, promesas de curas milagrosas, diagnósticos
Ejemplo de título efectivo:
"Nueva terapia reduce síntomas de artritis en un 70% según estudio clínico"
🏛️ 5.2. POLÍTICA

Tono de Voz

  • Objetivo e imparcial sin sesgo partidista
  • Analítico y crítico sin ser confrontacional
  • Formal pero accesible para diversos lectores
  • Equilibrado presentando múltiples perspectivas

Tipo de Lenguaje

  • Términos políticos explicados cuando sea necesario
  • Evitar eufemismos que oscurezcan la verdad
  • Preferir lenguaje concreto sobre abstracto
  • Citar textualmente a las fuentes para mantener objetividad

Estructura Recomendada

  1. Titular: Acción política o evento y sus actores principales
  2. Lead: El hecho político, quién lo realiza, cuándo, dónde, por qué es relevante
  3. Antecedentes: Contexto histórico y político del tema
  4. Posiciones: Perspectivas de diferentes actores políticos
  5. Análisis: Implicaciones políticas, sociales, económicas
  6. Voces múltiples: Citas de políticos, analistas, ciudadanos
  7. Próximos pasos: Qué sigue en el proceso político

Palabras Clave Frecuentes

gobierno, congreso, legislación, elecciones, política-pública, reforma, partido, candidato, votación, democracia, debate, proyecto-de-ley

Enfoque Editorial

  • Prioridad 1: Verificación rigurosa de hechos y declaraciones
  • Prioridad 2: Equilibrio entre diferentes posiciones
  • Prioridad 3: Contexto que ayude a entender el impacto
  • Evitar: Tomar partido, opiniones personales, rumores sin confirmar
Ejemplo de título efectivo:
"Congreso aprueba reforma tributaria con cambios para clase media"
⚠️ Importante: En noticias políticas, es crucial mantener la objetividad. Presenta hechos verificables y múltiples perspectivas sin favorecer ninguna posición.
⚽ 5.3. DEPORTES

Tono de Voz

  • Dinámico y energético reflejando la emoción del deporte
  • Apasionado pero profesional sin caer en el fanatismo
  • Narrativo contando historias humanas detrás del deporte
  • Inclusivo para aficionados casuales y expertos

Tipo de Lenguaje

  • Términos deportivos explicados si son específicos
  • Usar verbos de acción para crear dinamismo
  • Incluir estadísticas y datos relevantes
  • Expresiones coloquiales del deporte cuando sean apropiadas

Estructura Recomendada

  1. Titular: Resultado, logro o evento deportivo destacado
  2. Lead: El momento decisivo o resultado clave
  3. Desarrollo del evento: Cómo se desarrolló la competencia/partido
  4. Momentos clave: Jugadas, goles, puntos decisivos
  5. Contexto: Posiciones en tabla, historial, antecedentes
  6. Declaraciones: Citas de atletas, entrenadores, dirigentes
  7. Próximos desafíos: Siguientes partidos, competencias, objetivos

Palabras Clave Frecuentes

campeonato, victoria, derrota, entrenador, equipo, jugador, competencia, resultado, gol, punto, marca, récord, torneo, clasificación

Enfoque Editorial

  • Prioridad 1: Resultados y hechos objetivos
  • Prioridad 2: Historia humana y emocional del deporte
  • Prioridad 3: Estadísticas que den contexto
  • Evitar: Favoritismo exagerado, críticas destructivas a atletas
Ejemplo de título efectivo:
"Colombia clasifica al Mundial con goleada histórica 4-0 sobre Argentina"
💻 5.4. TECNOLOGÍA

Tono de Voz

  • Informativo y entusiasta sobre la innovación
  • Accesible para lectores no técnicos
  • Actualizado y moderno en el lenguaje
  • Práctico enfocado en el impacto real en las personas

Tipo de Lenguaje

  • Términos técnicos explicados con analogías cotidianas
  • Evitar jerga innecesaria (buzzwords)
  • Usar comparaciones para explicar especificaciones
  • Enfocarse en beneficios para el usuario, no solo características

Estructura Recomendada

  1. Titular: Innovación tecnológica y su principal beneficio
  2. Lead: Qué es la tecnología, quién la desarrolla, cuándo estará disponible
  3. Cómo funciona: Explicación técnica simplificada
  4. Características principales: Especificaciones relevantes
  5. Aplicaciones prácticas: Cómo impacta la vida diaria
  6. Contexto del mercado: Competencia, precio, disponibilidad
  7. Perspectiva: Tendencias futuras, evoluciones esperadas

Palabras Clave Frecuentes

tecnología, innovación, inteligencia-artificial, aplicación, software, hardware, desarrollo, lanzamiento, actualización, dispositivo, digital, startup

Enfoque Editorial

  • Prioridad 1: Beneficios reales para los usuarios
  • Prioridad 2: Explicación clara de conceptos complejos
  • Prioridad 3: Contexto de cómo encaja en tendencias actuales
  • Evitar: Exageraciones ("revolucionario"), jerga excesiva, tecnocentrismo
Ejemplo de título efectivo:
"Nueva IA de Google traduce conversaciones en tiempo real sin conexión a internet"
🎨 5.5. CULTURA

Tono de Voz

  • Sensible y respetuoso con expresiones culturales diversas
  • Apreciativo y crítico sin ser elitista
  • Descriptivo y evocador para transmitir experiencias
  • Inclusivo celebrando la diversidad cultural

Tipo de Lenguaje

  • Lenguaje rico y descriptivo para transmitir experiencias sensoriales
  • Términos artísticos explicados cuando sea necesario
  • Evitar snobismo o lenguaje excesivamente académico
  • Usar metáforas y comparaciones para describir arte

Estructura Recomendada

  1. Titular: Evento cultural, obra o expresión artística destacada
  2. Lead: El evento/obra, artista, cuándo, dónde, por qué es significativo
  3. Descripción: Elementos visuales, sonoros, conceptuales
  4. Contexto artístico: Movimiento, influencias, trayectoria del artista
  5. Impacto cultural: Significado para la comunidad/sociedad
  6. Voces: Citas de artistas, curadores, críticos, asistentes
  7. Información práctica: Cómo acceder, horarios, costos

Palabras Clave Frecuentes

arte, cultura, exposición, festival, artista, obra, museo, galería, patrimonio, teatro, música, literatura, tradición, expresión

Enfoque Editorial

  • Prioridad 1: Hacer el arte accesible a todos
  • Prioridad 2: Contextualizar la obra en su momento cultural
  • Prioridad 3: Celebrar la diversidad de expresiones
  • Evitar: Elitismo, juicios absolutos, imposición de gustos
Ejemplo de título efectivo:
"Botero regresa a Medellín con exposición gratuita de 50 obras inéditas"
💰 5.6. ECONOMÍA

Tono de Voz

  • Serio y autorizado en el manejo de cifras
  • Claro y didáctico al explicar conceptos complejos
  • Objetivo presentando datos sin sesgo
  • Práctico enfocado en el impacto real en las personas

Tipo de Lenguaje

  • Términos económicos explicados con ejemplos cotidianos
  • Cifras presentadas con contexto y comparaciones
  • Evitar jerga financiera innecesaria
  • Usar analogías para explicar conceptos macroeconómicos

Estructura Recomendada

  1. Titular: Hecho económico y su impacto principal
  2. Lead: El evento económico, cifras clave, quiénes se ven afectados
  3. Cifras y datos: Estadísticas relevantes con contexto
  4. Análisis: Qué significan estos números
  5. Causas: Factores que llevaron a esta situación
  6. Voces expertas: Citas de economistas, analistas, empresarios
  7. Impacto práctico: Cómo afecta a empresas, trabajadores, consumidores
  8. Proyecciones: Expectativas y escenarios futuros

Palabras Clave Frecuentes

economía, mercado, inflación, crecimiento, PIB, empresa, inversión, bolsa, finanzas, empleo, comercio, negocio, precio

Enfoque Editorial

  • Prioridad 1: Exactitud en cifras y datos
  • Prioridad 2: Explicar el impacto en la vida diaria
  • Prioridad 3: Contextualizar dentro de tendencias económicas
  • Evitar: Alarmismo, predicciones sin fundamento, sesgo ideológico
Ejemplo de título efectivo:
"Inflación en Colombia baja a 3.5% y alivia presión en el costo de vida"
🎓 5.7. EDUCACIÓN

Tono de Voz

  • Optimista y constructivo sobre el futuro educativo
  • Informativo y orientador para estudiantes y familias
  • Respetuoso con docentes y comunidad educativa
  • Crítico pero propositivo ante problemas educativos

Tipo de Lenguaje

  • Términos pedagógicos explicados de forma sencilla
  • Lenguaje inclusivo que abarque todas las modalidades educativas
  • Enfoque en beneficios para estudiantes y sociedad
  • Evitar tecnicismos académicos innecesarios

Estructura Recomendada

  1. Titular: Iniciativa educativa, política o innovación pedagógica
  2. Lead: El programa/cambio educativo, quién lo implementa, a quiénes beneficia
  3. Detalles del programa: Cómo funciona, qué incluye
  4. Beneficiarios: Cuántos estudiantes, niveles, regiones
  5. Contexto educativo: Situación actual, necesidades que atiende
  6. Voces educativas: Citas de docentes, estudiantes, rectores, expertos
  7. Impacto esperado: Objetivos, metas, transformación proyectada

Palabras Clave Frecuentes

educación, estudiante, docente, universidad, colegio, aprendizaje, enseñanza, pedagogía, becas, calidad-educativa, formación, academia

Enfoque Editorial

  • Prioridad 1: Información útil para la comunidad educativa
  • Prioridad 2: Destacar innovaciones y buenas prácticas
  • Prioridad 3: Visibilizar desafíos con perspectiva constructiva
  • Evitar: Comparaciones que estigmaticen, soluciones simplistas, elitismo
Ejemplo de título efectivo:
"Universidad Nacional ofrece 500 becas de pregrado para jóvenes de zonas rurales"
🌍 5.8. MEDIO AMBIENTE

Tono de Voz

  • Urgente pero no alarmista sobre desafíos ambientales
  • Esperanzador al presentar soluciones y acciones positivas
  • Científico y riguroso basado en evidencia
  • Motivador para inspirar acción individual y colectiva

Tipo de Lenguaje

  • Términos científicos explicados con ejemplos tangibles
  • Datos y cifras contextualizados en impacto real
  • Lenguaje que conecte problemas globales con acciones locales
  • Evitar catastrofismo paralizante

Estructura Recomendada

  1. Titular: Situación ambiental o acción ecológica destacada
  2. Lead: El evento/problema ambiental, ubicación, qué está en juego
  3. Dimensión del problema: Datos, cifras, extensión del impacto
  4. Causas: Factores que generan la situación
  5. Consecuencias: Impacto en ecosistemas, comunidades, futuro
  6. Soluciones: Iniciativas, proyectos, acciones en marcha
  7. Voces: Citas de ambientalistas, científicos, comunidades afectadas
  8. Llamado a la acción: Qué pueden hacer los lectores

Palabras Clave Frecuentes

medio-ambiente, cambio-climático, sostenibilidad, conservación, biodiversidad, energías-renovables, contaminación, reciclaje, ecosistema, naturaleza

Enfoque Editorial

  • Prioridad 1: Información científica verificada
  • Prioridad 2: Equilibrio entre problema y soluciones
  • Prioridad 3: Empoderar a los lectores para actuar
  • Evitar: Negacionismo, exageraciones, discurso derrotista
Ejemplo de título efectivo:
"Amazonas colombiano recupera 10.000 hectáreas con programa de reforestación comunitaria"
🎬 5.9. ENTRETENIMIENTO

Tono de Voz

  • Ligero y ameno pero profesional
  • Entusiasta compartiendo la pasión por el entretenimiento
  • Conversacional como una charla entre conocedores
  • Respetuoso con artistas y audiencias diversas

Tipo de Lenguaje

  • Lenguaje accesible y contemporáneo
  • Referencias culturales que conecten con la audiencia
  • Descripciones vívidas que transmitan la experiencia
  • Evitar spoilers innecesarios (advertir cuando haya)

Estructura Recomendada

  1. Titular: Estreno, evento o noticia del mundo del entretenimiento
  2. Lead: La noticia principal, artistas involucrados, fecha de lanzamiento/evento
  3. Detalles de la producción: Argumento (sin spoilers), género, duración
  4. Elenco/Artistas: Quiénes participan, trabajos previos relevantes
  5. Contexto: Antecedentes, expectativas, tendencias
  6. Reacciones: Primeras impresiones, críticas, audiencia
  7. Dónde ver/escuchar: Plataformas, cines, lugares de acceso

Palabras Clave Frecuentes

entretenimiento, película, serie, música, actor, artista, estreno, festival, concierto, streaming, cine, televisión, celebridad, taquilla

Enfoque Editorial

  • Prioridad 1: Informar sobre estrenos y eventos relevantes
  • Prioridad 2: Generar interés sin revelar demás
  • Prioridad 3: Celebrar la diversidad de gustos y expresiones
  • Evitar: Chismes sin fundamento, juicios personales severos, elitismo
Ejemplo de título efectivo:
"Shakira anuncia gira mundial con fecha confirmada en Colombia para julio"
🔬 5.10. CIENCIA

Tono de Voz

  • Riguroso y preciso en el manejo de información científica
  • Asombrado y curioso transmitiendo la fascinación por el descubrimiento
  • Didáctico explicando conceptos complejos de forma accesible
  • Objetivo evitando sensacionalismo

Tipo de Lenguaje

  • Términos científicos explicados con analogías comprensibles
  • Proceso científico descrito claramente (hipótesis, método, resultados)
  • Magnitudes presentadas con comparaciones tangibles
  • Evitar exageraciones sobre implicaciones de descubrimientos

Estructura Recomendada

  1. Titular: Descubrimiento científico y su importancia principal
  2. Lead: El hallazgo, quién lo hizo, dónde, por qué es significativo
  3. El descubrimiento: Qué encontraron/descubrieron
  4. Cómo lo hicieron: Metodología en términos accesibles
  5. Qué significa: Implicaciones científicas y prácticas
  6. Contexto científico: Cómo se relaciona con conocimiento previo
  7. Voces expertas: Citas de investigadores, científicos
  8. Próximos pasos: Investigaciones futuras, aplicaciones potenciales

Palabras Clave Frecuentes

ciencia, investigación, estudio, descubrimiento, experimento, científico, laboratorio, datos, análisis, teoría, evidencia, avance

Enfoque Editorial

  • Prioridad 1: Exactitud científica y verificación de fuentes
  • Prioridad 2: Traducción efectiva de conceptos complejos
  • Prioridad 3: Contexto de por qué el descubrimiento importa
  • Evitar: Sensacionalismo, afirmaciones no probadas, simplificación excesiva
Ejemplo de título efectivo:
"Investigadores colombianos desarrollan prueba que detecta cáncer en 30 minutos"
🔬 Nota Importante: En noticias científicas, siempre verifica que los estudios mencionados estén publicados en revistas revisadas por pares (peer-reviewed). Evita reportar estudios preliminares como conclusiones definitivas.

7. Instrucciones para Trabajo Colaborativo en GitHub

👥 Flujo de Trabajo Colaborativo

Antes de Comenzar

  1. Asegúrate de tener Git instalado en tu computadora
  2. Configura tu nombre y email en Git:
    git config --global user.name "Tu Nombre"
    git config --global user.email "tu@email.com"
  3. Clona el repositorio del proyecto

Durante el Ejercicio

1. Crea tu Rama Personal

git checkout -b tu-nombre-apellido

Esto crea una rama independiente donde trabajarás sin afectar el trabajo de otros.

2. Trabaja en tus Noticias

Crea y edita tus archivos HTML siguiendo las instrucciones del ejercicio.

3. Guarda tus Cambios Frecuentemente

git add .
git commit -m "Añadida noticia de salud sobre vacunas"

Haz commits cada vez que completes una noticia o un grupo de cambios relacionados.

4. Actualiza tu Rama

Periódicamente, actualiza tu rama con los cambios de main:

git pull origin main

5. Sube tus Cambios

git push origin tu-nombre-apellido

Al Finalizar

Crear un Pull Request

  1. Ve al repositorio en GitHub
  2. Haz clic en "Compare & pull request"
  3. Completa el formulario:
    • Título: "Noticias de [Tu Nombre] - Ejercicio Completo"
    • Descripción: Lista las 30 noticias creadas por categoría
  4. Asigna el pull request al profesor
  5. Haz clic en "Create pull request"

⚠️ Resolución de Problemas Comunes

Conflicto al hacer Push

Si Git te indica que hay conflictos:

  1. Actualiza tu rama: git pull origin main
  2. Abre los archivos con conflictos en tu editor
  3. Busca las marcas de conflicto (<<<<<<<, =======, >>>>>>>)
  4. Decide qué versión mantener o combina ambas
  5. Elimina las marcas de conflicto
  6. Guarda el archivo
  7. git add .
  8. git commit -m "Resueltos conflictos"
  9. git push origin tu-rama

Olvidaste Crear una Rama

Si comenzaste a trabajar en main por error:

  1. Guarda tus cambios sin hacer commit
  2. git stash (guarda cambios temporalmente)
  3. git checkout -b tu-nombre-apellido
  4. git stash pop (recupera tus cambios)
  5. Continúa trabajando normalmente

Quieres Deshacer el Último Commit

git reset --soft HEAD~1

Esto deshace el último commit pero mantiene tus cambios para que puedas corregirlos.

👍 Mejores Prácticas

  • Haz commits frecuentes: No esperes a terminar todo
  • Mensajes descriptivos: Explica qué cambios hiciste
  • Sincroniza regularmente: Pull de main cada 30 minutos
  • Revisa antes de push: Verifica que todo funcione
  • Comunica problemas: Pide ayuda si encuentras dificultades
